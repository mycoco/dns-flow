# dns-flow

[English](README.md) | 中文

dns-flow 是一款面向浏览器入口的网站访问分流工具。

它的设计目标是：让用户只需要给浏览器配置一个本地接入地址，就可以按网站、DNS 解析方式、解析后的 IP 结果，决定访问是直连、走代理、使用指定 DNS / DoH，还是直接拦截。

## 主要功能

- 网站访问分流：按域名规则匹配不同网站，为不同网站选择不同的访问策略。
- 多分流方案管理：可以创建多个分流方案，例如“默认分流”“办公网络分流”“测试环境分流”。
- 浏览器统一入口：每个分流方案提供一个本地监听地址和端口，浏览器只需要接入这个入口。
- DNS 分流：指定网站可以使用指定 DNS 服务器解析。
- DoH 分流：指定网站可以使用 DNS over HTTPS 解析，适合需要加密 DNS 查询的场景。
- 代理分流：指定网站可以转发到 HTTP / HTTPS 代理。
- 系统 DNS：指定网站可以回退到操作系统默认 DNS。
- 网站拦截：指定网站可以直接拦截，不继续访问。
- 解析后二次分流：先完成 DNS、DoH 或系统解析，再根据解析出来的 IP 所属国家/地区、CIDR 网段或地址范围决定直连、代理或拦截。
- 绑定物理网卡：规则命中后，如果目标地址属于局域网，可以优先通过指定的本地物理网卡访问，避免被全局代理、VPN 或虚拟网卡影响。
- hosts 映射：可以把指定域名固定到指定 IP，优先于普通 DNS 解析。
- 请求记录：查看访问命中了哪条规则、最终走向哪个目标，方便排查配置是否生效。
- 配置导入导出：可导出分流方案并导入到其他设备。
- 系统设置：支持语言、开机启动、日志、置顶、浏览器扩展接入、软件更新等配置。

## 分流是如何工作的

dns-flow 的核心思路是把复杂的访问策略收进一个浏览器入口中。

用户在 dns-flow 中创建一个分流方案，例如监听：

```text
127.0.0.1:5353
```

然后把浏览器或浏览器扩展配置为使用这个入口。之后浏览器访问网站时，dns-flow 会按规则判断这个域名应该走哪种策略。

一条网站访问规则通常包含：

- 指定网站：需要匹配的域名表达式，例如 `*google*`、`*.openai.com`。
- 分流动作：DNS、DoH、代理、系统 DNS 或拦截。
- 分流目标：对应的 DNS 地址、DoH 地址或代理地址。
- 优先级：数值越大越优先匹配。
- 备注：用于说明这条规则的用途。
- 启用状态：可以临时关闭规则而不删除。

### 基础分流

基础分流是在命中域名规则后直接决定访问策略。

常见例子：

- `*.example.com` 使用 `223.5.5.5` 解析。
- `*.openai.com` 使用 DoH 解析。
- `*test*` 走指定 HTTP 代理。
- `*ads*` 直接拦截。
- 未配置的网站继续使用系统 DNS。

### 解析后二次分流

有些网站仅靠域名无法判断最终访问路径，因为同一个域名可能解析到不同国家/地区或不同网段的 IP。此时可以使用“解析后二次分流”。

流程是：

1. 先用系统 DNS、指定 DNS 或 DoH 完成域名解析。
2. dns-flow 读取解析出的 IP。
3. 再按 IP 条件继续判断访问方式。

解析后条件支持：

- 国家/地区：例如中国、美国、日本、新加坡、香港等。
- CIDR 网段：例如 `8.8.8.0/24`。
- 地址范围：例如私有地址、公网地址等。

解析后的动作支持：

- 直连：直接使用本机网络访问。
- 代理：转交给指定代理。
- 拦截：阻止访问。

典型场景：

- 国内 IP 直连，国外 IP 走代理。
- 私有地址优先走本地网络，避免被全局代理或 VPN 影响。
- 某些国家/地区的解析结果直接拦截。

### 绑定物理网卡访问局域网

在一些网络环境中，电脑可能同时存在物理网卡、VPN、虚拟网卡、代理工具网卡等多个网络接口。如果全局代理或 VPN 接管了访问路径，访问路由器、NAS、打印机、内网服务等局域网地址时，可能会出现访问失败或绕路。

dns-flow 在网站访问规则的高级设置中提供“局域网访问时优先使用本地网络”能力。启用后，可以选择一个本地网络接口。当规则命中的目标最终属于局域网地址时，dns-flow 会优先使用选中的本地网络访问。

这个能力适合：

- 访问路由器管理页、NAS、内网 Git、内网测试服务。
- 开着全局代理或 VPN 时，仍希望局域网地址走真实物理网卡。
- 多网卡环境下，希望某些局域网访问固定走指定网卡。

为了降低误选概率，dns-flow 会优先展示正在运行、拥有 IPv4 地址的本地网络接口，并过滤常见虚拟网卡、回环接口、Docker、WSL、TAP/TUN、WireGuard、Tailscale、ZeroTier 等接口。

## 使用方法

### 1. 创建分流方案

打开 dns-flow 后，进入“网站访问分流”，点击“新建分流”。

建议先填写：

- 分流名称：例如“默认网站访问分流”。
- 监听地址：个人本机使用建议保持 `127.0.0.1`。
- 浏览器接入端口：例如 `5353`，确保端口没有被其他程序占用。
- 保存后立即启用：建议开启，保存后即可开始使用。

如果需要让局域网内其他设备访问，可以选择 `0.0.0.0`，但建议同时配置访问认证或来源 IP 限制。

### 2. 添加网站访问规则

在分流方案中点击“新建网站规则”。

常用配置方式：

- 让指定网站使用指定 DNS：选择“让指定网站使用指定 DNS”，填写域名规则和 DNS 地址。
- 让指定网站走代理：选择“让指定网站走代理”，填写域名规则和代理地址。
- 让指定网站使用加密 DNS：选择“让指定网站使用加密 DNS（DoH）”，填写 DoH 地址。
- 国内网站直连、国外网站走代理：选择对应场景，配置解析方式和国外代理。
- 高级分流规则：手动选择分流动作、目标、优先级、备注和解析后规则。

如果这条规则用于访问局域网服务，可以展开高级设置，开启“局域网访问时优先使用本地网络”，然后选择要绑定的本地物理网卡。

多个网站可以用英文分号分隔：

```text
*google*;*.openai.com;*github*
```

匹配规则支持通配符：

```text
*.example.com
*keyword*
example.com
```

### 3. 配置未命中规则的处理方式

如果一个网站没有命中任何规则，建议保持“其他网站按系统默认处理”开启。

这样未配置的网站会继续使用系统 DNS，不会因为没有规则而访问失败。

### 4. 启动分流

保存分流方案后，在“网站访问分流”列表中点击“启动”。

启动成功后，页面会显示运行状态和监听地址。浏览器只需要接入这个地址，就可以使用 dns-flow 的分流能力。

### 5. 配置浏览器入口

根据你的使用方式，将浏览器或浏览器扩展指向 dns-flow 的监听地址。

例如分流方案监听：

```text
127.0.0.1:5353
```

则浏览器相关配置应指向这个本地入口。之后浏览器访问网站时，dns-flow 会自动按规则分流。

### 6. 查看请求记录

进入“请求记录”页面，可以查看：

- 访问时间
- 访问域名
- 命中的分流方案
- 命中的规则
- 分流目标
- 访问状态

如果某个网站没有按预期走代理、DNS 或 DoH，可以先查看请求记录，确认它是否命中了正确的规则。

## 常见配置示例

### 指定网站使用阿里 DNS

```text
指定网站：*.example.com
分流动作：DNS
DNS 地址：223.5.5.5
```

### 指定网站使用 DoH

```text
指定网站：*.example.com
分流动作：DoH
DoH 地址：https://dns.google/dns-query
```

### 指定网站走代理

```text
指定网站：*.example.com
分流动作：代理
代理地址：http://127.0.0.1:7890
```

### 国内直连，国外走代理

```text
指定网站：*
先使用：系统 DNS / 指定 DNS / DoH
解析后规则：
  中国：直连
  其他国家/地区：代理 http://127.0.0.1:7890
```

### 局域网服务绑定物理网卡

```text
指定网站：nas.local;*.lan
分流动作：系统 DNS
高级设置：开启“局域网访问时优先使用本地网络”
选择本地网络：选择当前连接路由器的物理网卡
```

适合在开启全局代理或 VPN 时访问 NAS、路由器、打印机、内网系统等局域网资源。
